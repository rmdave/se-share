Yep, we want to show case the mashup capability of the Apigee Edge Platform

Bing Take an lat/long and search within 5 kilometres of that address for a hospital (or some other entity). You will need a Bing Map Key - http://www.microsoft.com/maps/create-a-bing-maps-key.aspx. I created a Basic key for the demo.
IPInfoDB account We assume that a browser made the invocation. We use this service to do a IP to Geo lookup and get the lat/long. You will need an account for IP Info DB - http://www.ipinfodb.com/
OpenWeather (no account needed) This demo uses APIs from the free OpenWeatherMap site - http://www.openweathermap.org/
Apigee account You will need API Services account; start here - http://apigee.com/about/

Here is what this demo will do:

  1. Allow user to submit a form with a radius
  2. Lookup their lat/long by calling an IP to Geo service
  3. Start an async call to get weather from OpenWeatherMap
  4. Call Bing Maps to get banks within the radius specified
  5. Mashup the Weather and Points of Interest data and return to caller
  6. Future considerations covered in another demo - mobile optimization for this user
API Services Feature Description
Key/Value Map Store configurations within the Apigee storage and not within policies. This allows you to promote policies from development to test without changes to hard coded keys/configurations.
Service Callout Policy A very quick way to perform a callout albeit a synchronous callout.
Async Callout Fire off an async call to an external service and continue the API processing flow.
Enrichment of the response Take the response from the 'backed' server (Bing Maps) and augment it with additional information (the weather data that was asynchronously requested).
Partner/Developer Management Manage who can access the APIs via configuration and zero disruption to the IT development team.
Custom Analytics Start capturing metrics on specific values within the request and easily display and analyze it without disrupting IT!
